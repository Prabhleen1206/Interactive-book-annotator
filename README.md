# Project Title
Interactive-book-annotator
A data-driven reading platform that allows users to annotate books, track activity, and generate insights using Python and SQLite.

## Problem Statement
Existing digital reading platforms primarily focus on passive content consumption and not content understanding and also do not provide an integrated environment for users to actively engage with texts through structured note-taking and data-driven insights. Additionally, there is a lack of simple systems that allow users to store, manage, and analyze their reading behavior and annotations efficiently using database-backed solutions.

## Proposed Solution
A web-based application that allows users to read public domain books,and understand content by simplifying complex paragraphs, creating and managing annotations, and store all interactions in a structured database. The system enables users to track their reading activity, retrieve notes, and gain insights using SQL-based queries, thereby transforming reading into an interactive and data-driven experience.

## Key Features
* Public Domain Book Display
* Comment Section with each paragraph
* Note Creation and Storage
* User Activity Tracking
* Search and Filter Notes
* Data Insights using SQL Queries

## Technical Approach
* Programming Language: Python
* Frontend: Streamlit
* Backend: Python (integrated with Streamlit)
* Database: SQLite
* Data Handling: SQL queries for storage, retrieval, and analysis

## Success Metrics
* Successful storage and retrieval of user notes
* Execution of at least 5 meaningful SQL queries
* Accurate data representation in query outputs
* User Engagement Time > 15 minutes/session
* Annotation Usage Rate > 60% of active users
* App Response Time < 3 seconds

## Timeline
Week 1: Requirement analysis + UI design + database schema design
Week 2: Development of reading interface and note-taking functionality
Week 3: Database integration + implementation of SQL queries
Week 4: Testing, debugging, and final deployment
